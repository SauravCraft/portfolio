PROJECT // 001
Networked Inventory System
Problem
Standard replication caused latency spikes in inventory updates during competitive multiplayer — unacceptable for real-time gameplay.
Architecture
Server-authoritative storage using Fast Array Serialization and Delta Replication. All mutations RPCed to server; client receives predicted state.
Implementation
Custom FFastArraySerializer subclass with per-item dirty flags. Item add/remove validated through server RPCs. Client UI updated via delegate broadcasts on rep notify.
Result
Significantly reduced network overhead. Item sync latency dropped, supporting 4-player concurrent inventory operations without bandwidth spikes.
